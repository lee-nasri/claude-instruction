# email-consumer

> This is an example knowledge file for **email-consumer**.
> In practice, this file contains the service overview, queue message contracts,
> email template rendering, provider integration (e.g. SendGrid, SES), retry
> and dead-letter queue strategy that Claude needs to understand before working on this service.
>
> **How to generate this for your own service:**
> 1. Open Claude Code inside your project directory
> 2. Run `/init` — Claude will read your codebase and generate a `CLAUDE.md`
> 3. Ask Claude: *"Summarize this into a knowledge file for ~/.claude/knowledge/"*
> 4. Save the output as `~/.claude/knowledge/email-consumer.md`
